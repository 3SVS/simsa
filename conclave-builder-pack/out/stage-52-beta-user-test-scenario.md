# Stage 52 — 베타 사용자 테스트 시나리오

실제 베타 사용자가 Conclave PR review workflow를 처음부터 끝까지 써보는 시나리오. 제품 사용자 관점.

대상: **AI로 PR을 만든 개발자 또는 PM.**

---

## 배경

요즘 많은 PR이 AI(코딩 에이전트)로 만들어집니다. 코드는 그럴듯하지만 "제품 설명서/요구사항 기준에 정말 맞는지"는 확신하기 어렵습니다. Conclave는 이 PR이 **제품 기준에 맞는지 확인**하고, 안 맞는 부분을 **수정 지시서**로 만들고, 수정 후 **다시 확인**해서 **좋아진 점을 PR에 공유**하는 흐름을 제공합니다.

## 사용자 목표

> "내(또는 AI)가 만든 PR이 우리 제품 설명서 기준에 맞는지 확인하고, 남은 문제만 골라 수정 지시서로 만들고, 고친 뒤 다시 확인해서 개선됐는지 비교하고, 그 결과를 팀에게 PR comment로 공유하고 싶다."

## 시작 조건

- GitHub 계정으로 Conclave dashboard에 로그인돼 있음.
- 확인하고 싶은 **public repo의 PR** 하나 (없으면 `3SVS/My-first-product` PR #1로 체험).
- 프로젝트에 "제품 설명서/확인 항목"이 설정돼 있음(없으면 데모 기본값).

---

## 수행 과제

### 과제 1 — PR 확인 결과를 이해한다
- 프로젝트 → GitHub → PR 선택 → "확인" 실행.
- 결과를 본다: 항목별로 **통과 / 안 맞음 / 확인 부족 / 결정 필요**.
- 관찰: 각 항목이 왜 그 상태인지(근거) 이해가 되는가?

### 과제 2 — 남은 문제를 찾는다
- 확인 기록(History) → 해당 run 상세.
- "이번에 다룰 항목"에 기본으로 **통과하지 않은 항목**이 골라져 있는지 본다.
- 관찰: "지금 신경 써야 할 게 무엇인지" 한눈에 들어오는가?

### 과제 3 — Fix Pack을 생성한다
- "남은 문제 Fix Pack" 또는 상세에서 "선택한 항목으로 Fix Pack 만들기".
- 생성된 **수정 지시서(Claude/Codex용)**를 복사한다.
- 관찰: 이 지시서를 그대로 코딩 에이전트에게 줘서 고칠 수 있을 것 같은가?

### 과제 4 — 수정 후 다시 확인한다
- (실제 또는 가정) 코드를 고친 뒤, "남은 문제 다시 확인"으로 같은 항목을 재확인한다.
- 관찰: 다시 확인이 "남은 문제만" 빠르게 도는가?

### 과제 5 — 이전 결과와 비교한다
- 다시 확인 후 새 기록에서 **"이전 확인 기록과 비교"**를 본다.
- `안 맞음 → 통과`처럼 무엇이 좋아졌고/남았고/새로 생겼는지 본다.
- 관찰: 개선 여부가 명확히 보이는가?

### 과제 6 — 비교 결과를 PR comment로 남긴다
- "이 비교 결과를 PR comment로 남기기" → preview 확인 → "GitHub에 남기기".
- GitHub PR에서 comment를 확인한다.
- 관찰: 팀원이 이 comment만 보고 "뭐가 나아졌는지" 이해할 수 있는가?

---

## 관찰 포인트 (진행자/기록자용)

- 각 과제에서 **멈칫하거나 헤맨 지점**(어디서, 왜).
- 버튼/문구를 **오해**한 곳.
- "이게 맞나?" 하고 **신뢰가 흔들린** 순간.
- 기대했는데 **없던 정보**.
- 과제 완료까지 걸린 **체감 시간**.

## 성공 기준

- 6개 과제를 **외부 도움 없이** 끝까지 수행.
- 사용자가 "AI PR을 확인 → 고칠 거리 파악 → 다시 확인 → 공유"의 가치를 스스로 말로 설명할 수 있음.
- 치명적 막힘(P0/P1) 없음.

## 질문 리스트 (테스트 후)

1. 이 도구가 해결해주는 문제를 한 문장으로 말한다면?
2. 가장 유용했던 단계와 가장 헷갈렸던 단계는?
3. PR 확인 결과(통과/안 맞음/확인 부족/결정 필요)를 신뢰할 수 있었나? 왜?
4. Fix Pack 지시서를 실제로 쓸 것 같은가?
5. 비교(이전 → 현재) 표시가 도움이 됐나?
6. PR comment로 공유하는 게 팀에 도움이 될까?
7. 다시 쓸 의향이 있는가? (1~5)
8. 빠졌다고 느낀 기능/정보가 있다면?
