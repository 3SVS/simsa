# Conclave 베타 — 현재 범위와 한계 (Known Limitations)

> 부정적 목록이 아니라 **"지금 베타에서 어디까지 되고, 무엇은 일부러 아직 안 한다"**를 분명히 하는 문서입니다.
> 테스트 전에 한 번 읽으면 "이건 버그가 아니라 설계"인 부분에서 시간을 아낄 수 있습니다.

## 지금 베타에서 잘 되는 것
- 공개(public) GitHub repo의 PR을 항목별로 확인.
- 확인 기록(history), 다시 확인, 이전과의 비교(상태 전환 포함).
- Fix Pack(수정 지시서) 생성.
- 결과를 PR 코멘트로 미리보기 + 게시.

## 현재 범위 / 한계

### 1. 비공개(private) repo는 아직 정식 지원이 아님
- 베타는 **공개 repo 기준**입니다. private repo full support는 다음 단계로 미뤘습니다(권한 범위를 신중히 넓히려고).
- 직접입력에 private repo를 넣으면 "비공개 저장소는 아직 지원하지 않아요"로 안내됩니다.

### 2. 조직(org) repo가 목록에 안 보일 수 있음 → 직접입력으로 해결
- GitHub 조직이 외부 앱 접근을 제한하면, 그 org의 repo가 **목록에 안 나옵니다**(GitHub 보안 정책 때문, 정상 동작).
- 이때는 저장소 화면의 **"목록에 없는 저장소 직접 입력"**에 `owner/repo`(예 `3SVS/My-first-product`)를 넣으면 연결됩니다(공개 repo 한정).
- 근본적으로 목록에 띄우려면, 그 조직 관리자가 Conclave 앱을 승인하면 됩니다(선택).

### 3. PR comment는 사용자가 확인 후 직접 게시
- 결과는 **미리보기**로 먼저 보여주고, **게시 버튼을 눌러야** PR에 올라갑니다. 자동으로 막 달지 않습니다.

### 4. Fix Pack은 "수정 지시서"이지 자동 코드 수정이 아님
- Fix Pack은 사람/AI(Claude·Codex)에게 넘길 **수정 가이드 + 프롬프트**입니다.
- Conclave가 코드를 직접 고치거나 커밋/브랜치를 만들지 **않습니다**(아래 6 참고).

### 5. 크레딧 실제 차감/차단은 꺼져 있음 (OFF)
- 베타 동안 `ENABLE_ACTUAL_CREDIT_DEBITS=false`, `ENABLE_CREDIT_BLOCKING=false`.
- 화면에 보이는 크레딧/허용량 안내는 **참고용 미리보기**이며, 실제로 돈/크레딧이 빠지지 않습니다.

### 6. 아직 하지 않는 것 (의도적으로 베타 범위 밖)
- **자동 코드 수정(autofix) / patch / commit / branch 생성** — 안 함.
- **GitHub status check**(PR 체크 표시) 생성 — 안 함.
- **결제(payment) 연동** — 안 함.
- **OAuth 권한 확대** — 안 함(공개 repo 읽기 + 코멘트 수준 유지).

## 그래서 베타에서 봐주셨으면 하는 것
- 위 범위 **안에서** workflow가 이해되고 신뢰가 가는지.
- 어디서 멈추는지, 어떤 문구가 헷갈리는지.
- "이 범위면 실제로 쓰겠다 / 안 쓰겠다"의 솔직한 판단.
