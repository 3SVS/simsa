# Conclave 베타 — 사용자 온보딩 안내

> 이 문서는 베타 테스터에게 전달하는 안내문입니다. GitHub를 쓸 줄 알면 개발자가 아니어도 그대로 따라할 수 있게 썼습니다.

## Conclave가 하는 일 (한 문장)

**GitHub Pull Request(PR)의 코드 변경이, 내가 만들려던 제품 요구사항을 실제로 충족하는지 자동으로 확인해주고, 그 결과를 PR에 코멘트로 남겨주는 도구**입니다.

쉽게: "이 PR이 우리가 약속한 걸 진짜 다 했나?"를 항목별로 **통과 / 안 맞음 / 확인 부족 / 결정 필요**로 정리해 줍니다.

## 베타의 목적

- 실제 GitHub PR로 review workflow가 **이해되고 쓸 만한지** 확인합니다.
- 막히는 지점, 헷갈리는 화면, 신뢰가 안 가는 부분을 찾습니다.
- 새 기능 요청보다 **"이게 실제로 도움이 되나"**를 봅니다.

## 대상 사용자

- GitHub에 repo와 PR이 있는 분 (개인 프로젝트도 OK).
- PM / 기획자 / 개발자 누구나. 코드를 깊게 안 봐도 결과는 읽을 수 있습니다.

## 준비물

1. **GitHub 계정** (PR을 만들 수 있는 권한).
2. **공개(public) repo** 하나와 그 안의 **열린 PR 하나**.
   - 비공개(private) repo는 아직 베타 범위가 아닙니다(아래 "직접입력"과 known limitations 참고).
3. 웹 브라우저 (크롬 권장).
4. 약 **15~20분**.

### GitHub repo/PR이 없다면 (3분 준비)

1. GitHub에서 **New repository** → public으로 생성.
2. 아무 파일이나 추가하고 새 브랜치에서 한 줄 고친 뒤 **Pull Request**를 엽니다.
3. 그 PR 번호(#1 등)를 기억해 두세요.
   - 빠르게 보고 싶으면 데모 repo `3SVS/My-first-product`의 PR #1을 그대로 써도 됩니다(공개).

## 접속

- **Dashboard**: https://conclave-dashboard.vercel.app
- 로그인 페이지는 없습니다. 접속하면 바로 시작할 수 있고, 브라우저별로 내 작업이 저장됩니다(같은 브라우저로 다시 오면 이어집니다).

## GitHub 연결 방법

1. 프로젝트를 하나 만든 뒤, **설정(Settings)** 화면으로 갑니다.
2. **"GitHub로 연결"** 버튼 → GitHub 인증 화면에서 **Authorize**.
3. 다시 dashboard로 돌아오면 "✓ GitHub 계정이 연결됐어요"가 보입니다.
   - 권한은 **공개 repo 읽기 + 코멘트** 수준입니다. 코드를 바꾸거나 푸시하지 않습니다.

## repo가 목록에 안 보일 때 (조직/org repo)

GitHub **조직(org) 소속 repo**는 보안 설정 때문에 목록에 안 나올 수 있습니다. 이때는:

1. 저장소 선택 화면 아래 **"목록에 없는 저장소 직접 입력"** 칸에
2. **`owner/repo`** 형식으로 입력합니다. 예: `3SVS/My-first-product`
3. **연결**을 누르면 바로 연결됩니다. (공개 repo만 됩니다.)

- 익숙해지면 PR 하나당 2~3분.
