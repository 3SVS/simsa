# Conclave 베타 — 테스트 가이드 (사용자용 과제)

> 위에서 아래로 순서대로 따라 하세요. 각 과제는 **할 일 / 기대 결과 / 실패하면 기록할 것 / 관찰 포인트**로 되어 있습니다.
> 막히면 멈추지 말고 그 칸의 "실패하면 기록할 것"만 적어두고 다음으로 넘어가도 됩니다.

- Dashboard: https://conclave-dashboard.vercel.app
- 데모로 빠르게 보려면: repo `3SVS/My-first-product`, PR `#1` (공개)

---

## 과제 1. Dashboard 접속
- **할 일**: 브라우저에서 dashboard URL 접속.
- **기대 결과**: 프로젝트 목록 화면이 열린다(빈 목록이어도 정상). 404/빈 화면 아님.
- **실패하면 기록**: URL, 화면 캡처, 브라우저 콘솔(F12) 에러.
- **관찰 포인트**: 첫인상에 "여기서 뭘 하는 곳인지" 감이 오는가?

## 과제 2. GitHub 연결
- **할 일**: 프로젝트 생성 후 설정 → "GitHub로 연결" → Authorize.
- **기대 결과**: dashboard로 복귀 + "GitHub 계정이 연결됐어요" 표시. 주소가 `conclave-dashboard.vercel.app`로 돌아옴.
- **실패하면 기록**: 돌아온 주소, 화면 메시지, 콘솔/네트워크 에러.
- **관찰 포인트**: 권한 요청 범위가 불안하지 않았는가?

## 과제 3. 프로젝트 만들기
- **할 일**: "새 프로젝트" → 아이디어/제품 설명 입력 → "저장하고 프로젝트 시작하기".
- **기대 결과**: 프로젝트 개요 화면으로 이동(404 아님).
- **실패하면 기록**: 입력값, 이동한 주소, 에러.
- **관찰 포인트**: 무엇을 입력해야 하는지 분명했는가?

## 과제 4. repo 연결
- **할 일**: 설정에서 저장소 선택. 목록에 있으면 클릭, 없으면 **직접입력**(`owner/repo`, 예 `3SVS/My-first-product`).
- **기대 결과**: 저장소가 연결됨("저장소가 연결됐어요").
- **실패하면 기록**: 목록에 내 repo가 보였는지 / 직접입력 시 입력값과 메시지.
- **관찰 포인트**: 내 org repo가 목록에 보였는가, 직접입력이 필요했는가?

## 과제 5. PR 선택
- **할 일**: GitHub 화면에서 PR 목록 불러오기 → 테스트할 PR(예 #1) 선택/연결.
- **기대 결과**: PR이 목록에 뜨고 선택된다.
- **실패하면 기록**: PR이 안 보임 / 선택 안 됨 + 메시지.
- **관찰 포인트**: 어떤 PR을 골라야 할지 알기 쉬웠는가?

## 과제 6. PR 확인(Review) 실행
- **할 일**: 선택한 PR에 대해 "확인하기" 실행.
- **기대 결과**: 잠시 후 결과가 항목별로 표시(통과/안 맞음/확인 부족/결정 필요 개수).
- **실패하면 기록**: 로딩이 멈춤 / 에러 메시지 / 너무 오래 걸림(몇 초였는지).
- **관찰 포인트**: 기다리는 동안 "지금 뭐 하는 중"인지 알 수 있었는가?

## 과제 7. 확인 결과 이해
- **할 일**: 결과 항목을 하나씩 읽기.
- **기대 결과**: 각 항목의 상태와 이유, "다음 단계"가 이해된다.
- **실패하면 기록**: 어떤 라벨/문구가 헷갈렸는지 구체적으로.
- **관찰 포인트**: **통과 / 안 맞음 / 확인 부족 / 결정 필요**의 뜻이 직관적이었는가?

## 과제 8. 남은 문제 Fix Pack 만들기
- **할 일**: "남은 문제 Fix Pack" 생성 → 내용/복사/다운로드 확인.
- **기대 결과**: 수정 지시서(Claude/Codex용 프롬프트 + 파일들)가 생성됨.
- **실패하면 기록**: 생성 실패 / 빈 내용 / 복사·다운로드 안 됨.
- **관찰 포인트**: 이 지시서를 실제로 개발(혹은 AI)에 넘길 만했는가?

## 과제 9. 다시 확인하기 (quick re-run)
- **할 일**: "남은 문제 다시 확인" 실행.
- **기대 결과**: 새 확인 결과가 생기고, 이전 기록과 연결됨.
- **실패하면 기록**: 새 확인이 안 생김 / 같은 화면 / 에러.
- **관찰 포인트**: "다시 확인"이 뭘 다시 보는지 분명했는가?

## 과제 10. 이전 결과와 비교하기
- **할 일**: 다시 확인 후 자동으로 뜨는 **비교** 보기.
- **기대 결과**: 좋아진 항목 / 아직 남은 항목 / 새로 생긴 문제 / 변화 없음, 그리고 상태 전환(예: 안 맞음 → 통과)이 보인다.
- **실패하면 기록**: 비교가 안 뜸 / 숫자가 이상함.
- **관찰 포인트**: "무엇이 나아졌는지" 한눈에 들어왔는가?

## 과제 11. PR comment preview 또는 게시
- **할 일**: "이 결과를 PR comment로 남기기" → **미리보기** 확인 → (원하면) **게시**.
- **기대 결과**: 미리보기에 결과/비교가 한국어로 정리됨. 게시하면 GitHub PR에 코멘트가 달린다.
- **실패하면 기록**: 미리보기 깨짐 / 게시 실패 / GitHub에 안 보임.
- **관찰 포인트**: 이 코멘트를 **팀에 보여줄 만큼 신뢰**가 갔는가? (안 가면 어디가 약했는지)
