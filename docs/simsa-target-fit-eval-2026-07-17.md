# Simsa 타겟 적합성 실측 — 비개발자에게 실제로 유용한 결과물인가 (2026-07-17)

> Bae 지시: "비개발자 노코드/바이브코더/초보 AI유저에게 실제로 원하는 결과물을 내주는지
> 아주 객관적인 평가. 다양한 분야 적합성 + vercel/supabase/cloudflare/resend/sentry만
> 제안하는 건 아닌지 + codex/replit/lovable 등 다양한 곳에서 작업 가능한 산출물인지."
> 방법: 프로덕션 익명 실측(관대한 해석 금지). 러너: `tools/simsa-target-fit-eval/run.mjs`,
> 원자료: 같은 폴더 `dump-*.json`.

## 방법

서로 다른 10개 분야의 타겟-현실적 아이디어(반찬가게 예약 / 초등 받아쓰기 / 교회 소모임 /
필라테스 수강권 / 웹툰 포트폴리오 / 농산물 꾸러미 / solo 반려견 기록 / 원룸 수리접수 /
시니어 복지관(70대) / 아이폰 3D 게임[대조군])를 프로덕션 `idea-to-spec-draft` →
`export-builder-pack(target=both)`에 투입. 정량 지표 + 산출물 정독.

## 결과 A — 분야 적합성: **강함 (실측 최고 성적)**

- **10/10 실제 LLM** (mock fallback 0), 케이스당 질문 6개·항목 10개·빌더팩 7파일.
- **템플릿 반복도 0.000** — 45쌍 전수 교차에서 아이디어 간 항목 제목 중복이 단 1건도 없음.
  전부 그 분야의 언어로 생성됨.
- 질문의 실질성(정독 판정): 반찬가게→"예약 주문 관리는 별도 화면/문자/카톡 중 뭐가 편한가",
  시니어→"큰 글씨·타이핑 최소화·중복 신청 처리·로그인 없이 시작", 원룸→사진 첨부·처리 상태.
  답에 따라 제품이 실제로 달라지는 질문들 = PRD가 요구한 "좋은 질문" 기준 충족.
- **가드 전부 유효(재검증)**: solo 케이스 권한류 질문 0개 · 네이티브 게임 케이스 정직 경고
  발화 · 개발자 용어 누수 질문/openQuestions **0건**(#351 당일 배포분이 10케이스에서 유지).

### 발견된 흠 (LLM 질문 품질, P2)

필라테스 케이스에서 3건:
1. "회원 조회 화면을 **모바일 앱(iOS/Android)으로도 만들지**" — 개발 AI가 만들 수 없는
   네이티브 앱을 선택지로 제시(실현가능성 정직성과 모순). 프롬프트에 "네이티브 앱을 선택지로
   제안 금지" 규칙 없음.
2. "시ンpleSheet처럼" — 깨진 토큰(모지바케성 출력). 후처리 검출 없음.
3. "데이터는 **어디에 저장**하고 싶으신가요" — 비개발자가 결정할 수 없는 저장 위치 질문
   (openQuestions는 #351이 걸러주지만 **questions 배열은 후처리 미적용**).

## 결과 B — 스택 편향: **초안 깨끗 / 빌더팩은 Vercel·Supabase 단일 경로**

- **스펙 초안(질문·항목·결정)**: 10/10에서 스택 토큰(15종) **0회** — 아이디어 단계는
  도구 중립이 완전히 지켜짐.
- **빌더팩**: 10/10 전부에서 Vercel(5회/팩)·Supabase 언급. 실체는 강제가 아니라
  `export.ts`의 **하드코딩 온보딩 예시**("자주 쓰는 서비스 예시: Supabase 가입→키 복사,
  Vercel 로그인→Deploy")와 배포 지시("Vercel·GitHub의 MCP 또는 CLI") — 즉 스택을 지정하진
  않지만 **유일하게 제시되는 워크드 예시가 항상 같아** 사실상의 기본 경로. cloudflare/
  resend/sentry는 등장 0 (Bae 우려 중 이 셋은 실측상 없음).
- 아이디어별 필요에 따른 예시 변주(예: 이메일 필요 앱에 이메일 서비스 예시)는 **없음** —
  결정론 템플릿이므로 당연한 결과.

## 결과 C — 도구 이식성: **구조적 갭 (가장 큰 발견)**

- `ExportTarget = "claude_code" | "codex" | "both"` — **빌더팩 산출 형태가 2종뿐.**
  `built-with.ts`는 10개 도구(v0/lovable/bolt/cursor/replit/windsurf/codex…)를 알지만
  이는 **캡처 태그일 뿐**, 산출물 분기가 아님.
- 빌더팩의 CLI 전제 토큰: `.env.local`·"터미널"·localhost·MCP·CLI·"붙여넣으세요" —
  **10/10 전부 등장.** 파일 트리 + 로컬 실행 + git/MCP 배포를 전제.
- **Lovable/Replit/v0/Bolt 같은 웹빌더**(타겟 유저가 가장 많이 쓰는 부류)는 파일시스템
  없이 채팅 프롬프트로 움직이고 배포가 내장 — 현재 팩의 `.env.local 만들기`, `MCP 연결`,
  `로컬 서버 실행` 지시는 그 환경에서 **실행 불가능한 지시**가 됨. 프롬프트 텍스트 자체는
  복사해 넣을 수 있으나, 지시의 절반이 환경에 안 맞음.
