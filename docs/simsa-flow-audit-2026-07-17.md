# Simsa 전체 플로우 E2E 감사 — 탭 구조·프로세스 효율성·적합성 (2026-07-17)

> Bae 지시: "지금 플로우 전체적인 검수로 우리 플랫폼 내에서 탭별 이동 및 전체 프로세스
> e2e의 효율성과 적합성을 다시 검토하고 평가해줘." 방법: 프로덕션에서 아이디어 위저드를
> 실제 완주(실 LLM, proj_hepck0xm)하고 프로젝트의 전 탭을 순회 측정.
> 러너: `tools/simsa-completion-loop-spike/flow-audit.mjs`, 원자료: `flow-audit-result.json`
> + `flow-audit-shots/*.png` (22장).

## A. 효율성 — 측정값

| 구간 | 측정 | 판정 |
|---|---|---|
| 아이디어 → 프로젝트 생성 | 위저드 4단계(아이디어→이해→질문→완성), 사람 기준 최소 **약 11클릭**(생성1 + 질문 추천 6 + 도구선택 1 + 단계진행 3) + LLM 대기 2회(각 20~40초) | 대기 안내 정직("보통 20~30초 걸려요 — 창을 닫지 마세요"), 단계 바 명확. **질문 6개를 개별 클릭**이 최대 마찰 |
| 프로젝트 → 빌더팩 손에 쥐기 | 사이드바 1클릭 + 에이전트 선택 1클릭(#356) + 복사/zip 1클릭 = **3클릭** | 짧음. 복귀 훅("다 만드셨나요? 돌아와서 연결하세요")도 페이지 안에 존재 |
| 탭 수 | 사이드바 8개(개요/아이디어/제품 설명서/확인 항목/빌더 팩/확인 결과/시각 검수/코드 저장소/연결) + 심사 지도 | 스테이지 그룹(준비→검수→결과·수정→고급→언제든지)이 있어 방향감 양호 |
| 에러/데드엔드 | 전 탭 `errorish` 마커 0, 빈 화면 0 | 이전 세션들(#320/#331)의 수정이 유지됨 |

**효율 개선 여지 1순위: 질문 일괄 수락.** "잘 모르겠어요, 추천대로 할게요"를 질문마다
눌러야 함(6클릭). 상단에 **"모두 추천대로 하고 넘어가기" 1클릭** 옵션이면 초심자 경로가
11→6클릭으로 줄어든다. (개별 오버라이드는 그대로 유지)

## B. 적합성 — 발견 (우선순위순)

1. **[P1] 개요 '지금 할 일'이 아이디어 갈래에 코드 갈래 할 일을 그대로 제시.** 방금 아이디어로
   만든 프로젝트의 개요 카드가 "2. **코드 저장소 연결 — AI가 만든 코드가 있는 GitHub 저장소를
   연결하세요** / 3. 확인 실행 — 코드 변경을 골라…"라고 지시. #327/#328이 걸음(walk)과 탭은
   갈래별로 정리했지만 이 카드 카피가 남음. 아이디어 갈래의 다음 행동은 "빌더 팩 받기 → 개발
   AI로 만들기 → 완성되면 주소 넣고 검수"여야 한다. (스크린샷 02-project-landing)
2. **[P1] 심사 지도 페이지 = 내부 용어 통째 누수.** "인테이크", "제품 브리프", "릴리스
   체크포인트", "머지", "배포", "검증 중", "계획 및 인테이크" — 개발자/PM 내부 개념이 비개발자
   화면에 그대로. 제품 전체가 지키는 비개발자 언어 원칙(#351 등)을 이 페이지만 위반.
   (스크린샷 10-tab-10-map)
3. **[P2] '개선 참여' 팝업이 위저드를 가림.** 첫 LLM 대기 중 등장해 마지막 질문의 선택지와
   하단 진행 버튼을 물리적으로 덮음(스크린샷 01-wizard-r6). 노출 시점을 프로젝트 생성 완료
   후로 지연하고 위저드 화면에서는 띄우지 않아야.
4. **[P2] 스테이지명 '검수' 아래에 '빌더 팩'.** 빌더팩은 만들기(생성) 단계인데 스테이지
   라벨이 검수 — 이름과 내용 불일치. 스테이지명 재검토(예: 준비 → 만들기 → 검수·수정).
5. **[P3] 이해 단계 주요 흐름 이중 번호.** "1. 1. 강사가…" — LLM 문자열 선두 번호와 UI 목록
   번호 중복. 렌더에서 선두 `N.` 스트립 1줄. (스크린샷 01-wizard-r4)
6. **[P3] 아이디어 갈래에 코드 흔적 잔존.** checks 페이지의 "코드 확인 (GitHub)" 섹션,
   export 페이지 내 `/github` 링크 — #328은 탭만 숨겼고 페이지 내부 섹션/링크는 남음.
7. **[P3] 페이지 제목·라벨 정리.** 시각 검수/연결 페이지는 h1 부재. '코드 저장소'(settings)
   페이지 제목이 "코드 저장소 연결"인데 실제 내용은 서비스 준비 + 배포 + 알림 설정 혼합 —
   서비스 준비를 찾으려는 사용자가 이 라벨로 못 찾음. '코드 저장소'와 '연결' 두 사이드바
   라벨의 구분도 모호.
