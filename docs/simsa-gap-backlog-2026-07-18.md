# Simsa 격차 백로그 (2026-07-18, Bae "1~8 모두 작업" 승인)

계획된 스테이지 밖에서 식별된 격차의 단일 추적 문서. 제품 루프(아이디어→팩→만들기→
복귀 검수→수정→수용) 기준. 각 항목은 착수 시 필요하면 별도 설계(D/RC 방식)로 확장한다.
완료 판정은 ship-guard(머지·배포·라이브 실증) 기준.

## 1차 식별 (Bae 승인: 전부 작업)

| # | 항목 | 왜 | 규모 |
|---|---|---|---|
| G1 | **복귀 이메일 루프** | 팩 받고 나간 유저를 되부르는 장치 0 — 복귀율이 활성화 지표 그 자체. 기존 email 채널(Resend·opt-in) 재사용 + 팩 다운로드 시 이메일 수집(선택) | 중 |
| G2 | **막힘 도우미** | 비개발자의 실패 지점은 만들기 도중(에이전트 에러·키 발급). 에러 붙여넣기→쉬운 말 다음 행동. 복귀 접점 겸용 | 중 |
| G3 | **검수 이력 비교(회귀 감지)** | "지난번엔 통과, 이번에 깨짐"을 시스템이 말해주기. run 데이터는 이미 저장됨 — 비교 로직+화면만 부재 | 중 |
| G4 | **시각 검수 깊이** | ①저장→새로고침 지속성(Potemkin 최종시험, 최우선) ②로그인 뒤 화면(테스트 계정 전달 UX) ③모바일 뷰포트 ④권한 팝업 | 대(분할) |
| G5 | **합의 레이어 적용면 확대** | panel/council이 스펙 체크에만 — PR 코드 리뷰는 단일 모델 | 중 |
| G6 | **결제 실배선** | 플랜 게이팅은 있는데 유료가 될 길(가격→체크아웃→ls_subscriptions)이 없음. LS 골격 존재. 단가는 llm_usage 실측으로 | 중 |
| G7 | **퍼널 계측 화면** | usage_events는 쌓이는데 방문→생성→팩→복귀→수용 퍼널을 보는 admin 뷰 부재 | 소–중 |
| G8 | **데이터 내구성** | localStorage 중심(checkResults·fixSuggestions·userProfile) — 기기 변경=유실. userKey에 유료 자격 묶임 리스크 포함. 로그인 승격과 한 축 | 대 |
