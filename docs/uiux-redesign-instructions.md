## 1. 개요(지휘센터)에 컴팩트 진행 표시

**문제:** 프로젝트를 열었을 때 "이 프로젝트가 어디까지 왔나"가 본문에서 바로 안 보임.
좌측 사이드바 진행지도는 네비게이션이고, 본문에는 진행 맥락이 없음.

**작업:**
- 개요(지휘센터) 화면에 3스텝 + 현재 위치를 보여주는 **컴팩트 진행 표시** 추가 —
  기존 `computeProjectSteps` 상태머신 재사용(새 로직 만들지 말 것).
- 배치는 지휘센터 CTA 카드 위 또는 우측. **상단 가로 타임라인은 지양** — 잠금
  힌트·하위 정보 들어갈 공간이 없고 스크롤 시 사라짐. 우측 세로 또는 CTA 위 컴팩트 권장.
- 사이드바와 중복이 아니라 "본문에서 바로 보이는 축약판" 역할. nextProjectAction
  CTA와 함께 "지금 여기 + 다음 이것"이 한눈에.

## 2. 확인결과 요약 재배치 — "3초 파악" 기준

**문제:** 확인결과 화면에서 요약과 상세가 같은 시각적 무게로 나열됨.

**기준:** 유저가 화면 도착 후 **3초 안에** — (a) "몇 개 통과 / 몇 개 문제" (b) "그래서
다음에 뭘 하면 되나".

**작업:**
- 요약(통과 N / 확인 필요 M — 숫자와 상태색)을 상단에 크게. 문장 설명은 최소화.
- 항목별 상세·근거·diff는 요약 아래 계층으로(접힘 또는 시각적으로 물러남).
- "다음 행동"(수정 지시서 받기 / 재검수 등)이 요약과 함께 바로 보이게.

## 3. 확인 항목 "+N개 더" 인라인 펼침

**문제:** "꼭 들어가야 할 것 +4개 더" 같은 표시가 클릭해서 펼쳐지지 않음.

**작업:**
- "+N개 더" 클릭 시 **그 자리에서 인라인 확장/접힘**. 별도 전체보기 페이지 만들지 말 것.
- 기본 노출 기준은 **개수가 아니라 중요도**: 필수 항목은 전부 노출, 부가 항목만 접힘.
  "필수인데 접혀서 못 본" 사고 방지.

## 4. 피드백 채널: mailto 제거 → 인앱 피드백 폼

**상태: 핵심 완료.** FeedbackModal이 backend `sendFeedback`(workspace-feedback,
migration 0058) 사용, mailto 제거됨. beta-feedback mailto 빌더는 미참조 dead code.

지시서 원안의 세부(추후 확인/보강):
1. ~~mailto 링크 제거~~ ✅
2. **인앱 피드백 모달**: 유형(버그/질문/제안) + 텍스트 + 보내기. ✅

## 5. 화면별 primary CTA 위계 ★ 최상위

**문제:** "시각 검수", "첫 검수 실행하기" 같은 핵심 실행 버튼이 눈에 안 들어옴. 유저가
도착해서 "아, 이걸 누르면 되겠구나"가 즉시 판단 안 됨. **활성화 이벤트로 가는 마지막
관문이 묻혀 있음 — 흐름이 마지막 1미터에서 끊김.**

**원칙:** 화면마다 "주인공 버튼"은 하나.
- primary 액션 **하나만** 크게, 채운(filled) 색, 콘텐츠 중앙 흐름에.
- 나머지 액션은 전부 물러남(outline/ghost/텍스트 링크).
- ⚠️ **함정**: "잘 보이게"는 크기가 아니라 **대비**. 다 키우면 다시 아무것도 안 보임.
  "버튼 키우기"가 아니라 "**화면별 위계 재정렬**".

**작업:**
1. 각 실행 화면(검수 실행, 시각 검수 등)에서 핵심 실행 버튼을 primary로 재정렬 —
   기준: 처음 온 유저가 도착 후 **생각 없이** 다음 행동 판단 가능한가.
2. 버튼 문구는 행동+결과형("첫 검수 실행하기") 유지·통일. 전문어 금지.
3. nextProjectAction(개요 CTA)과 각 화면 primary가 **같은 행동**을 가리키는지 일관성.
4. **전 화면 전수 적용** — 일부만 하고 끝내지 말 것.
