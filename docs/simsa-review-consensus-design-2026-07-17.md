# Simsa 검수 합의 레이어 설계 — 검증 패널(A) + 협의체(B), 플랜 2티어

2026-07-17, Bae 지시: "A, B를 전부 넣되, 유저가 무료버전은 A / 유료 유저는 B로 선택할 수
있게 해야지." 이 문서는 그 지시의 설계 잠금 문서다. 구현은 `design lock approved` 이후,
트레인별 `train N start approved`로만 착수한다.

**배경.** 현재 Simsa 검수는 단일 파이프라인(코드/스펙 체크 1개 모델 + 시각 검수 컨테이너 +
결정론 판정 사다리)이다. Bae가 정의한 제품 루프 "유저의 아이디어 → AI의 도움 →
**에이전트들끼리의 조율을 통한 검수** → 실제 개발 이후 다시 체크"에서 ③이 격차다.
정확도 원칙(2026-07-17 확립: 오탐 수정은 신호/잡음 분리로)과 정합하는 해법은 "판정을
관대하게"가 아니라 **틀리면 아픈 판정에 독립 시선을 추가**하는 것이다.

## 결정

### RC-1 — 2티어 구조 [LOCKED(승인 대기)]

- **A. 검증 패널(무료 포함 전원)**: 유해-판정(사용자에게 "안 맞음/작동 안 함"이라고
  말하게 되는 판정)에만 두 번째 모델의 독립 교차 확인을 붙인다.
- **B. 협의체(유료 선택)**: 복수 에이전트가 독립 소견 → 상호 반박 → 종합하는 풀 합의
  검수. 유료 유저가 검수 실행 시 **선택**한다(강제 아님, 기본은 A).
- 무료=A 고정, 유료=A(기본)/B(선택). 서버가 자격을 집행한다(UI 숨김만으로 게이팅 금지).

### RC-2 — A 검증 패널 설계 [PILOT]

- **트리거**: `failed`(안 맞음) 판정과 Potemkin류 "작동 안 함" 문구를 만들 판정만.
  `passed`/`inconclusive`는 교차 확인하지 않는다(비용, 그리고 원칙: 후하게 만드는 게
  아니라 틀린 유죄판결을 막는 장치다).
- **2차 시선**: 교차-벤더 우선 — OpenAI(`OPENAI_API_KEY` Worker 시크릿 존재) 판정
  프롬프트로 동일 증거를 독립 평가. OpenAI 불가 시 Anthropic 상위 모델로 폴백.
- **합의 규칙**: ①양쪽 동의 → 판정 확정(리포트에 "2중 확인됨" 표기) ②불일치 → 판정을
  "확인 필요"로 강등하고 두 관점을 모두 쉬운 말로 표기(단정 금지) ③2차 시선 호출 실패 →
  원판정 유지 + "단일 확인" 표기(fail-open이되 정직 — 조용히 2중인 척 금지).
- **파라미터(잠그지 않음)**: 교차 확인 대상 판정 상한 per 검수(초안 5), 타임아웃(초안
  15s), 2차 모델명.

### RC-3 — B 협의체 설계 [PILOT]

- **Worker-native로 새로 작성한다.** `packages/core`의 council(Mastra 그래프)은 Node
  전제라 재사용하지 않는다(2026-07-08 CEO 결정: 7층=Worker≠Node). 개념(독립 소견 →
  반박 라운드 → 종합)만 가져온다.
- **구조**: ①1라운드 — 가용 모델 3벤더(Anthropic/OpenAI/Gemini — 키 3종 Worker 시크릿
  존재)가 동일 증거로 **독립** 소견(서로의 답을 보지 않음) ②불일치 항목만 2라운드 —
  각자 타 소견에 대한 반박/동의 ③종합 — 합의/불일치/남은 위험을 비개발자 언어 리포트로.
  라운드 상한 2(하드), 항목 상한 per 검수(초안 10).
- **적용 면**: 스펙/코드 체크(check-draft)와 시각 검수 판정문 모두. 시각 검수의 증거
  수집(컨테이너)은 그대로 1회 — 합의는 **증거 해석**에만 붙는다(브라우저 N회 아님).
- **키 부재 시**: 가용 벤더 2개면 2자 협의로 강등+표기, 1개면 B 불가를 정직하게 안내하고
  A로 대체(조용한 대체 금지).

### RC-4 — 플랜 게이팅 [PILOT]

- **자격 해석(서버, 검수 요청 시)**: ①`ls_subscriptions`에 해당 `user_key`의
  `status='active'` 구독 존재 → paid ②베타 수동 그랜트 테이블(`plan_grants`, 신규 —
  admin 엔드포인트로 부여/회수) → paid ③그 외 → free.
- **요청 형태**: 검수 요청에 `reviewMode: "panel" | "council"`(기본 `panel`). free가
  `council` 요청 → 402 + 쉬운 안내(오류 아님). UI는 B 옵션을 잠금 배지로 노출(숨기지
  않음 — 존재를 알아야 업그레이드 동기가 됨).
- 결제 라이브 전 파일럿은 수동 그랜트로만. 과금 단가는 RC-7.

### RC-5 — 비용·한도 [PILOT]

- A: 검수당 추가 호출 ≤5(트리거 상한). B: 검수당 호출 ≤ 3벤더×2라운드+종합 = 7.
- 기존 beta_limits(검수 100/일) 산정에 모드별 가중 반영은 [OPEN] — 초기엔 동일 카운트.
- 사용량은 기존 `anthropic_usage` 로그 패턴을 벤더 일반화해 기록(단가 실측 근거).

### RC-6 — 도입 전 실측 게이트 [LOCKED(승인 대기)]

- 구현 완료 ≠ 배포. **픽스처 6종 + 실앱 3종으로 A/B 각각 오탐·미탐을 베이스라인(현
  단일)과 비교 실측**하고, 수치가 베이스라인 이상일 때만 배포 게이트를 연다. 결과는
  `docs/simsa-inspection-accuracy-eval-*.md`에 추가 기록(측정 기록 보존 원칙).

### RC-7 — B 과금 단가 [OPEN]

- 재검토 트리거: 결제 라이브 결정 시. 그 전까지 B는 수동 그랜트 파일럿 전용.
