## 휴먼 QA 체크리스트 (Bae 유저 시점, 예상 30~40분)

> 방법: **시크릿 창**(신규 유저 상태)에서 https://app.trysimsa.com 진입.
> 각 항목에서 막히거나(진행 불가), 속거나(오도), 다음 행동이 없으면(데드엔드) = **P0 기록**.
> 어색함/거슬림은 P1/P2로 메모만 하고 계속 진행.

### A. 아이디어 갈래 (본인의 진짜 아이디어로 — ② 재확인 겸)
- [ ] 갈래 선택 화면에서 세 문 중 뭘 고를지 3초 안에 이해되는가
- [ ] 본인의 실제 아이디어 입력(고유 디테일 포함) → 인터뷰 질문이 그 아이디어에 맞는가
- [ ] 생성된 제품 설명서·확인 항목에 **입력한 고유 디테일이 반영**됐는가 (② 판정)
- [ ] 준비 화면에서 다음 할 일이 보이는가 · 빌더 팩 다운로드가 자연스러운가

### B. 기존-앱 갈래 (아무 배포 URL로)
- [ ] 스텝1 질문("꼭 작동해야 하는 것" 포함)이 부담 없이 답할 만한가
- [ ] 생성 직후 랜딩(준비와 연결)에서 여기가 어디고 뭘 할지 아는가
- [ ] 개요의 "지금 할 일"이 이 갈래에 맞는가 (코드 미연결이면 연결 안내)
- [ ] checks에서 화면의 주인공 버튼이 하나로 읽히는가

### C. 검수 → 결과 → "왜"
- [ ] 시각 검수 실행 → 진행 표시 → 리포트가 비개발자 언어인가
- [ ] **"왜 이 판정인가요?" 펼침(신설)**: 3열 체인이 이해되는가 · 점수 없는 상태 라벨이 납득되는가 · "Simsa의 해석 — 측정된 사실 아님" 구분이 보이는가
- [ ] [고치기] → PR이 열리고 본문이 내 언어(KO)인가
- [ ] 공유 링크 생성 → 받은 사람 시점에서 열어보기

### D. 횡단 확인
- [ ] EN 토글 → A·B 입구 화면에 한국어 잔재가 없는가
- [ ] 모바일 폭(브라우저 창 360px)에서 갈래 선택·개요가 깨지지 않는가
- [ ] 아무 화면에서나 뒤로/전체 프로젝트로 탈출 가능한가

### 판정 기록
- P0 발견 시: 화면·행동·기대와 실제를 한 줄씩. **P0=0이면 ③·④ 동시 통과.**
- 발견된 P1/P2는 실행계획 백로그로 이관(오픈 차단 아님).
